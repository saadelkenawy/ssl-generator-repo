Installation

⚡ Quick Install via DNF/YUM Repository

# Step 1 — Add the repository
sudo curl -o /etc/yum.repos.d/ssl-generator.repo \
  https://saadelkenawy.github.io/ssl-generator-repo/ssl-generator.repo

# Step 2 — Install
sudo dnf install ssl-generator    # RHEL 8+ / Rocky / AlmaLinux / Fedora
sudo yum install ssl-generator    # RHEL 7 / CentOS 7
📦

Direct RPM Install (offline / air-gapped)

sudo rpm -ivh \
  https://saadelkenawy.github.io/ssl-generator-repo/x86_64/ssl-generator-1.0.0-1.x86_64.rpm

02

Mode 1 — Full PKI Generation

🏗

What it generates

  • →

    Root CA (4096-bit, 10-year validity)

    The trust anchor. Install root_cert.pem into your trusted store.
  • →

    Intermediate CA (4096-bit, 5-year validity)

    Signs all server certificates. Keep inter_key.pem secret.
  • →

    Server Certificate (2048-bit, 3-year validity)

    Ready-to-use TLS cert with SAN support.
▶

Run Mode 1

ssl-generator

Mode 2 — Additional Server Certificates

🔁

When to use Mode 2

Use Mode 2 when you already have a Root CA + Intermediate CA from a previous Mode 1 run, and you need to issue additional server certificates signed by the same CA — without rebuilding the entire PKI.

▶

Run Mode 2

ssl-generator
Enter choice (1 or 2): 2

Mode 2: Generate additional server certificate using existing CA
You will need the directory containing the existing CA files:
  - inter_key.pem   (Intermediate CA private key)
  - inter_cert.pem (Intermediate CA certificate)
  - root_cert.pem   (Root CA certificate)

Enter the full path to the directory containing CA files: /root/certs_test-com_20260325/
⚠ Always use the absolute path Use /root/certs_test-com_20260325/ — not a relative path like certs_test-com_20260325/ — to avoid path resolution errors.
📝

Interactive Prompts — Mode 2

  • 01

    CA directory path

    Full absolute path to directory containing inter_key.pem, inter_cert.pem, root_cert.pem
  • 02

    DN fields auto-extracted from existing intermediate cert

    C, ST, L, O, OU are reused automatically — no re-entry needed
  • 03

    New server domain (CN)

    e.g., proxy.test.com · api.test.com · Must contain a dot
  • 04

    SAN DNS entries (optional)

    Comma-separated, wildcards allowed
  • 05

    SAN IP entries (optional)

    Comma-separated IPv4 addresses
  • 06

    Create another certificate? (y/n)

    Loop to issue multiple certs in one session
